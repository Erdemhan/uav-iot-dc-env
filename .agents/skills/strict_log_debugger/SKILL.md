# Strict Log Debugger Skill

## Objective
Prevent speculative code edits and ensure all bug fixes are strictly justified by actual empirical log evidence.

## Mandatory Procedure on Process/Script Failure

1. **STOP & DO NOT EDIT CODE:**
   Do NOT touch or modify any source code files when a process fails (e.g., exit code 1 or FAILED status).

2. **INSPECT LOGS FIRST:**
   - Locate the exact log file generated by the failed process (e.g., `artifacts/scenario_runs/<timestamp>/S1-A/ppo.log`).
   - Read the exact log lines (e.g., `tail -n 50 <log_file>` or `grep -i "Traceback"`).
   - If the log file is on a remote or lab machine that you cannot read directly, ask the user to share the output using `cat <log_file>` or `tail -n 50 <log_file>`.

3. **VERIFY ROOT CAUSE:**
   - Identify the exact exception class, error line, and module from the log traceback.
   - Match the code line in the repository with the exact line in the stack trace.

4. **APPLY TARGETED FIX:**
   - Only after verifying the exact root cause from the log evidence, proceed to modify code.
   - Run a test command to verify that the error no longer occurs.
